本文介绍如何在 Hexo 博客中创建「分类（categories）」「标签（tags）」页面，以及如何扩展出一个「友情链接」等自定义页面。内容基于 Butterfly 主题实测整理。

一、Hexo 原生动态页面机制

Hexo 本身内置 4 种可自动更新的动态页面：

页面类型 是否自动更新 说明
index（首页） 是 自动列出文章（分页）
archives（归档页） 是 自动按时间归档
categories（分类页） 是 自动根据文章头部的分类生成
tags（标签页） 是 自动根据文章头部的标签生成

这四类页面不需要你手动维护内容，Hexo 在生成站点时会自动扫描所有文章的 Front Matter（头部注解）并动态更新。

六、总结对比

页面类型 来源 是否自动更新 更新方式
index / archives / categories / tags Hexo 内置生成器 自动 文章变化时自动生成
自定义静态页（如 links、about） 用户创建 手动 编辑 index.md
自定义带模板页 主题或插件 自动 从 _data/ 或逻辑渲染

七、推荐实践

  • 分类 / 标签 / 归档页 → 用 type: 自动生成
  • 友情链接 / 导航页 → 用 _data/*.yml 文件维护
  • 特殊展示页（项目展示、作品集） → 自定义 layout 模板

这样既能保持自动化，又能保持整洁可维护的结构。
