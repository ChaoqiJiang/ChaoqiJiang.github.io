数据查询

select 基本结构和基本查询语句

select 结构

select [ all | distnct ] <目标列表达式> [，<目标列表达式> ]...... from <表名或视图名> [，<表名或视图名>]..... [ where <条件表达式> ] [ group by <列名1> [ having <条件表达式> ] ] [ order by <列名2> [ asc | desc ] ];

聚集函数

COUNT（[ DISTINCT | ALL ] <列名>）
SUM （[ DISTINCT | ALL ] <列名>）
AVG （[ DISTINCT | ALL ] <列名>）
MAX （[ DISTINCT | ALL ] <列名>）
MIN （[ DISTINCT | ALL ] <列名>）

默认为all，如果想要去除对应列的重复项，可以使用distinct关键字。

简单查询

  1. 查询指定列

    • 查询表中的姓名与年龄信息

      select sname, sage from student;

      • 将要查询的列名写在select后面即可。

  2. 查询全部

    • 查询student表中所有的信息

      select * from student;

      • 当要查询所有列的时候，可以使用 * 替代当前表中所有列。

  3. 条件查询

    • 查询student表中男生的信息

      select * from student where sex = '男';

      • 使用where关键字进行筛选，注意此处只能是表达式，不可以是聚类函数。

  4. 去重查询

    • 查询student表中都有哪些系别

      select distinct dept as 系别 from student

      • 要对哪个属性进行去重，就在属性名前面加上distinct关键字。

  5. 对属性值起别名

    • 查询表中的学号和姓名信息，并对查询结果的字段名起别名为学号和姓名。

      select distinct sno as 学号, sname 姓名 from student ;

      • 属性 as 别名
      • 或者直接 属性名 别名

  6. 排序

    • 查询表中的姓名和性别，并按照性别进行降序排序。

      select distinct sname ,sex from student order by sex desc;

      • 关键字order by

  7. 聚类函数

    • 统计
      • 查询选修了课程的学生人数
        SELECT COUNT(DISTINCT Sno) FROM SC ;
    • 求平均
      • 计算1号课程的学生平均成绩
        SELECT AVG(Grade) FROM SC WHERE no=‘1’ ;
    • 计算最大值
      • 查询选修1号课程的学生最高分数
        SELECT MAX(Grade) FROM SC WHERE Cno=‘1’ ;

  8. 分组查询

    • 查询student表中每个系别都有多少人。

      select dept 系别,count(*) from student group by dept ;

      • 可以使用group by对查询的结果进行分类。

    • 利用having与聚类函数合并使用：查询选修了3门课程以上的学生学号。

      select sno from group by sno having count(*) > 3;

      • 需要注意的是，having必须和group by组合使用，对于要用聚类函数进行筛选的问题，由于where的局限性，可以转换成使用having的形式使用。

连接查询

​ 连接查询即在两个或两个以上表中联合查询相关信息，由于涉及到多个表，因此在对列进行操作时，应该写成：<表名> . <列名>。通常连接查询是在WHERE子句中，进行多个表中列的比较，从而得到相应的结果。

等值与非等值连接查询

WHERE子句的格式

<表名1> . <列名1> <比较运算符> <表名2> . <列名2>--连接条件（连接谓词）

  • 当比较运算符是“=”时，是等值连接
  • 当比较运算符是其它时，是非等值连接。如：

​ >、<、>=、<=、!=（<>）、BETWEEN...AND

样例：查询每个学生及其选修课程的信息

select student.*, sc.* from student, sc where student.sno = sc.sno;

查询结果没有去重，等价于关系代数中的等值连接。如果需要实现自然连接（去掉重复的属性列），则需要手动输入所需的目标列。

自身连接查询

样例
查询每门课程的间接先修课

SELECT C1.Cno, C2.Cpno FROM Course C1, Course C2 WHERE C1.Cpno = C2.Cno;

查询所有比刘伟工资高的教师姓名、工资和刘伟的工资

T(Tno,TN,Sex,Age,Prof,Sal,Comm,Dept)

S(Sno,SN,Sex,Age,Dept)

C(Cno,CN,CT)

TC(Tno,Cno)

SC(Sno,Cno,Score)

Select T1.TN,T1.Sal,T2.Sal From T T1,T T2 Where T1.Sal>T2.Sal and T2.TN=‘刘伟’

复合条件连接查询

查询选修2号课程并且成绩在90分以上的所有学生学号和姓名.

select sjkstudent.sno, sname from sjkstudent,sjksc where sjkstudent.sno = sjksc.sno and sjksc.cno = '2' and sjksc.grade > 90;

多表连接查询

查询每个学生的学号、姓名、选修课程名和成绩

SELECT Student.Sno,Sname,Cname,Grade FROM Student,Course,SC WHERE Student.Sno = SC.Sno AND SC.Cno = Course.Cno ;

嵌套查询

在一个SELECT语句中嵌入另一个SELECT语句

嵌入方式：
SELECT ... FROM ... WHERE ...(SELECT ... FROM ... WHERE ...);
前一个SELECT称为外层查询或父查询，WHERE里的SELECT称为内层查询或子查询。

子查询中不允许出现ORDER BY子句

允许多层嵌套，层层嵌套方式反映了SQL语言的结构化。

可分为：

不相关子查询
子查询的查询条件不依赖于父查询
相关子查询
子查询的查询条件依赖于父查询

不相关子查询

是由里向外逐层求解。即每个子查询在上一级查询处理之前求解，子查询的结果用于建立其父查询的查找条件。

相关子查询
  • 首先取外层查询中表的第一个元组，根据它与内层查询相关的属性值处理内层查询，若WHERE子句返回值为真，则取此元组放入结果表；
  • 然后再取外层表的下一个元组；
  • 重复这一过程，直到外层表全部检查完为止。
带IN谓词的子查询

查询与“张三”在同一个系学习的学生信息

SELECT * FROM Student WHERE Sdept IN ( SELECT Sdept ROM Student WHERE Sname = ‘张三’)；

查询选修了“数据库系统原理”的学生学号和姓名

SELECT Sno,Sname FROM Student WHERE Sno IN (SELECT Sno FROM SC WHERE Cno IN (SELECT Cno FROM Course WHERE Cname = ‘数据库系统原理’))；

带比较运算符的子查询

当子查询的结果确定是单个数值时，可以用比较运算符连接查询。

查询与“张三”在同一个系学习的学生信息

SELECT * FROM Student WHERE Sdept = (SELECT Sdept FROM Student WHERE Sname = ‘张三’ );

找出每个学生超过他平均成绩的课程号

  • 首先找出一个学生
  • 计算该学生的平均成绩
  • 找出超过这个平均成绩的课程号
  • 重复上面的3个步骤

SELECT Sno,Cno FROM SC x WHERE Grade >= ( SELECT AVG(Grade) FROM SC y WHERE y.Sno = x.Sno);

在子查询中的条件与父查询有关，这种查询称为相关子查询

带ANY或ALL谓词的子查询

用聚集函数实现
SELECT Sname,Sage FROM Student WHERE Sage < (SELECT MAX(Sage) FROM Student WHERE Sdept= ‘CS ') AND Sdept <> 'CS’;

查询其他系中比计算机科学系所有学生年龄都小的学生姓名及年龄。

方法一：用ALL谓词

SELECT Sname,Sage FROM Student WHERE Sage < ALL (SELECT Sage FROM Student WHERE Sdept= 'CS ') AND Sdept <> 'CS ’;

方法二：用聚集函数

SELECT Sname,Sage FROM Student WHERE Sage < (SELECT MIN(Sage) FROM Student WHERE Sdept= 'CS') AND Sdept <>'CS’;

带EXISTS谓词的子查询
  1. EXISTS谓词
    存在量词

带有EXISTS谓词的子查询不返回任何数据，只产生逻辑真值“true”或逻辑假值“false”。

若内层查询结果非空，则外层的WHERE子句返回真值；

若内层查询结果为空，则外层的WHERE子句返回假值；

由EXISTS引出的子查询，其目标列表达式通常都用* ，因为带EXISTS的子查询只返回真值或假值，给出列名无实际意义。

  1. NOT EXISTS谓词

若内层查询结果非空，则外层的WHERE子句返回假值

若内层查询结果为空，则外层的WHERE子句返回真值

样例：

​ 查询所有选修了1号课程的学生姓名。

嵌套查询：

select sname from student where ( from sc where sno = student.sno and cno = ‘1’);

连接运算:
SELECT Sname FROM Student, SC WHERE Student.Sno=SC.Sno AND SC.Cno= '1';

集合查询

多个SELECT查询的结果可以进行集合操作。

用于集合操作的谓词：

  • 并：UNION
  • 交：INTERSECT
  • 差：EXCEPT

​ 进行集合操作的各个查询结果应该是具有相同的列数，并且对应的数据类型相同

样例：

查询计算机系和年龄不大于19的学生
SELECT * FROM Student WHERE Sdept=‘CS’ UNION SELECT * FROM Student WHERE Sage <= 19；

等效于：SELECT * FROM Student WHERE Sdept=‘CS’ OR Sage <= 19；

小结

WHERE子句的格式

  • <属性列名> θ <属性列名>
  • <属性列名> θ <常量>
  • <属性列名> θ [ ALL | ANY ] （SELECT语句）
    • θ表示各种条件运算符
  • <属性列名> [ NOT ] LIKE <匹配串>
  • <属性列名> IS [ NOT ] NULL
  • [ NOT ] EXISTS （SELECT 语句）

数据更新

插入数据

插入元组

一次只能插入一条元组

基本格式

INSERT INTO <表名> [(<属性列名1> [，<属性列名2> ... ])] VALUES (<常量1> [，<常量2> ] ... );

说明

  • 在<表名>中插入一个新的元组（根据dbms不同插入的位置不同，如在oracle中插入的位置在最后）；
  • <属性列1>对应的数值是<常量1>，依次类推；
  • 如果缺少某一个属性列名，新的元组在该列上取值为空；
  • 如果没有任何属性列名，则VALUES中必须对应每个属性列都有值；

样例：将一个学生插入到student表中。

INSERT INTO Student(Sno,Sname,Ssex,Sdept,Sage) VALUES (‘010’,‘宋七’,‘男’, ‘CS’, 19 ) ;

也可以这样：

INSERT INTO Student VALUES (‘010’,‘赵九’,‘女’,18,‘CS’) ;

插入子查询结果

一次可以插入多条元组

基本格式

INSERT INTO <表名> [ (<属性列名1> [,<属性列名2> ... ])] <SELECT子查询语句>

说明

  • 在<表名>中插入、由<SELECT子查询语句>查询出来的一组元组

样例

建立新表，存放经过计算后得到的每个系学生的平均年龄。

CREATE TABLE Deptage (Sdept CHAR(15), Avgage SMALLINT ) ;

INSERT INTO Deptage(Sdept,Avgage) SELECT Sdept,AVG(Sage) FROM Student GROUP BY Sdept ;

修改数据

基本格式

UPDATE <表名> SET <列名> = <表达式> [，<列名> = <表达式> ]...[WHERE <条件表达式> ]

说明

  • 修改数据也称为更新数据
  • 将满足<条件表达式>的元组，用新的表达式结果代替相应属性列的值
  • 如果没有<条件表达式>，更新所有元组

样例

将所有学生的年龄加1：

update student set sage = sage + 1;

删除数据

基本格式

DELETE FROM <表名> [ WHERE <条件表达式> ]

说明

  • 将满足<条件表达式>的元组从表中删除
  • 如果没有<条件表达式>，则删除表中所有元组
  • 只会删除表中的元组，要删除整个表，只能使用DROP语句。

样例

删除学号为‘001’的学生信息：

DELETE FROM Student WHERE Sno = ‘001’
